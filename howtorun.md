# How to Run

## Data Flow Overview

The system processes data in two distinct pipelines: a real-time flow and a batch processing flow. Both flows originate from the same data source and converge in Grafana for visualization.

1. **Data Ingestion**
   Data is generated and sent to Kafka by the `kafka/producer.py` script.

2. **Real-time Flow (Streaming)**
   - The stream processing is handled by **Bytewax**, which is configured to autorun.
   - Bytewax consumes the streaming data from Kafka, processes it, and pushes it directly to **InfluxDB**.

3. **Batch Flow (Periodic)**
   - An **Airflow** DAG runs every 30 minutes to manage the batch data pipeline search for the edge_device_medallion_30m DAG.
   - This process takes the data, transforms it as needed, and pushes it into **MinIO** object storage.
   - **Trino** is configured to connect to MinIO, providing a SQL interface to query the batch data.

4. **Visualization**
   **Grafana** acts as the unified visualization layer, connecting to two different data sources to generate charts:
   - **InfluxDB**: Queried for real-time data and metrics.
   - **Trino (via MinIO)**: Queried for historical/batch data.

## How to View the Data

You can inspect the data at every stage of the pipeline using the provided web interfaces. Ensure your Docker containers are running before accessing these URLs.

### 1. Kafka (Raw Event Data)
- **URL**: [http://localhost:8088](http://localhost:8088)
- **Credentials**: None required.
- **How to view**: Open the Kafka UI, navigate to the **Topics** tab, select the `edge_computer_stats` topic, and click on **Messages** to view the live stream of JSON events generated by the producer.

### 2. InfluxDB (Real-time Processed Data)
- **URL**: [http://localhost:8086](http://localhost:8086)
- **Username**: `admin`
- **Password**: `adminpassword`
- **How to view**: Navigate to the **Data Explorer** (graph icon on the left menu). Select the `kafka_events` bucket to query and visualize the real-time metrics pushed by Bytewax.

### 3. Airflow (Batch Pipeline Monitoring)
- **URL**: [http://localhost:8082](http://localhost:8082)
- **Username**: `airflow`
- **Password**: `airflow`
- **How to view**: Check the DAGs view to monitor the status of the batch pipeline tasks. You can trigger DAGs manually and check task logs to see data processing details.

### 4. MinIO (Batch Storage)
- **URL**: [http://localhost:9001](http://localhost:9001)
- **Username**: `minioadmin`
- **Password**: `minioadmin`
- **How to view**: Access the Object Browser to inspect the raw-data, processed-data, or delta-lake buckets where Airflow stores the batch output files (e.g., Parquet or JSON).

### 5. Trino (SQL Query Interface for Batch Data)
- **URL**: [http://localhost:8080](http://localhost:8080)
- **Credentials**: Typically none or `admin`.
- **How to view**: This UI shows active and historical queries. To query the data in MinIO, you would typically connect to Trino on `localhost:8080` using a database tool like DBeaver or the Trino CLI, selecting the `iceberg` catalog.

### 6. Grafana (Unified Dashboards)
- **URL**: [http://localhost:3000](http://localhost:3000)
- **Username**: `admin`
- **Password**: `adminpassword`
- **How to view**: Go to **Dashboards** to view or create visualizations. The data sources for InfluxDB (real-time) and Trino (batch) are pre-provisioned. You can query both simultaneously to compare live and historical metrics.
